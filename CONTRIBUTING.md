# Інструкція з розробки (Contributing)

Дякую за інтерес до проєкту! Наразі я розробляю цей проєкт самостійно, але завжди радий конструктивному фідбеку.

## Як допомогти
1. **Знайшли помилку?** Створіть [Issue](https://github.com) з описом проблеми та кодом, який її викликає.
2. **Є ідея?** Опишіть її в розділі Issues для обговорення.

## Стек технологій
- **Мова:** MicroPython 
- **Залізо:** (наприклад, ESP32 / Raspberry Pi Pico)
- **Інструменти:** Thonny / mpremote / rshell

## Вимоги до коду
Якщо ви хочете запропонувати зміни (Pull Request):
- Дотримуйтесь стилю PEP 8 (з урахуванням обмежень MicroPython).
- Пишіть коментарі до складних частин коду.
- Переконайтеся, що код не займає занадто багато RAM (пам'ятайте про мікроконтролери).

---
*Примітка: Це сольний проєкт, тому відповідь на Pull Request може зайняти деякий час.*
